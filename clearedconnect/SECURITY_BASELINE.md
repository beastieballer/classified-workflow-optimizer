# ClearedConnect Security Baseline

## Data Types
- Lead/contact info (Sensitive)
- Project scope metadata (Sensitive/Restricted)
- Candidate/clearance-linked data (Restricted)

## Required Controls
- MFA on CRM, inbox, and collaboration tools
- Access by role only (Sales, Ops, Founder)
- Intake tagging: every submission labeled by classification
- No Restricted data in public inboxes or casual chat tools
- Quarterly access review + immediate offboarding revocation

## Transparency Controls
- Track major staffing decisions and handoffs
- Maintain evidence of access reviews and policy updates
- Preserve audit-ready change history for customer-facing workflows
